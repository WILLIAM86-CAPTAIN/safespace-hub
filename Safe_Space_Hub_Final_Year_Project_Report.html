ABSTRACT

This project report describes the design and implementation of Safe Space Hub, a web-based mental wellness support platform developed to help users access reliable screening tools, mental health resources, crisis support, and private self-care tracking.

The system resolves the problem of fragmented mental wellness support by providing a modern, accessible digital hub that centralizes mood tracking, screening assessments, crisis information, and recovery resources in one place.

The application was built using HTML5, CSS3, and JavaScript for the frontend, with a Node.js and Express backend. User data is managed through MongoDB Atlas and local browser storage for fallback access.

6

The Safe Space Hub supports validated screening tools such as PHQ-9 and GAD-7, presents curated educational resources, and offers quick emergency access to crisis hotlines. It includes an adaptive dashboard, secure login flow, and accessibility options for users who need a calm interface or larger text.

Testing covered local development and live deployment, with user feedback showing strong ease-of-use and improved confidence in finding support quickly. The report uses simple language so the project can be explained clearly during presentation.

CHAPTER 1 INTRODUCTION & BACKGROUND

1.1 Problem Statement

The Safe Space Hub project aims to solve the challenge of mental wellness support being scattered, hard to access, and not easy to manage. Users often find it difficult to locate validated screening tools, crisis hotlines, and practical recovery resources from one place.

Existing mental health resources in the local environment rely on paper records, phone recommendations, or web pages that are not friendly for people who need calm, simple access. This makes the experience stressful for people seeking help.

13

1.2 Previous Work & Methods

Before this project, mental health support was shared through print booklets, phone messages, and individual counselor referrals. There was no single online hub that offered both screening tools and crisis support in a safe, easy format.

Several existing wellness apps and platforms were reviewed, but they either required subscriptions, had poor local access, or did not include the simple crisis hotline connections needed for the project.

14

1.3 Background Context

The Safe Space Hub serves people who want fast mental wellness support, self-check tools, and helpful guidance without complex navigation. It is designed for users in Uganda and similar settings, where mobile-friendly web access is more common than native apps.

The system includes a dashboard, assessment tools, learning resources, and crisis hotline links. This makes it suitable for students, workers, first-time users, and family members looking for trusted support information.

15

1.4 Project Description

The Safe Space Hub is a responsive web application with user login, adaptive interface options, validated assessment tools, crisis contacts, and educational resources. It combines a lightweight frontend with a backend API for secure data handling.

The system was built to work in modern browsers, with strong accessibility features such as high contrast mode, large text, and calm visual settings for sensitive users.

16

1.5 Objectives & Justification

1.5.1 Main Objective

To create a digital mental wellness support platform that makes high-quality screening, crisis information, and resources easy to access for users in a safe online environment.

1.5.2 Specific Objectives

  • Build a secure login experience for users and protect session data.
  • Implement validated screening tools such as PHQ-9 and GAD-7.
  • Provide a dashboard for quick access to assessments, resources, and crisis contacts.
  • Use MongoDB Atlas for reliable cloud data storage and localStorage for client fallback.
  • Make the interface responsive and accessible for different devices.
  • Include a calm mode and large text for sensitive users.
17

1.5.3 Justification

The project is justified by the need for a single, easy-to-use platform that supports mental wellness and crisis access. It reduces the time users spend searching for resources and ensures that validated screening tools are simple to use and understand.

The Safe Space Hub also helps keep information organized and available without forcing users to install a native app or deal with a complex system.

3.2 Reliability

The system uses clear validation and error handling to make sure user actions do not fail silently.

3.2.1 Error Detection

  • Login fields are checked before submission.
  • Assessment answers are validated so incomplete forms are not sent.
  • Backend requests use try/catch to handle network errors.

3.2.2 Error Recovery

  • When the backend is unreachable, the browser displays a message and keeps the form state.
  • LocalStorage stores session keys so users can return to the dashboard after refresh.
  • Validation messages are shown below fields so users can fix errors quickly.

3.2.3 Acceptable Failure Rate

The target uptime for the deployed app is 99.5% with a reliable MongoDB Atlas service and hosted server.

3.3 Security

3.3.1 Hardware Security

The backend is hosted in cloud infrastructure with physical protection and controlled access. The system is designed to run on secure server hardware or cloud hosting.

3.3.2 Software Security

  • Login credentials are compared securely on the server.
  • JSON Web Tokens are used to secure authenticated sessions.
  • Express middleware handles JSON body parsing and CORS policies.
  • Password hashing is implemented using bcryptjs for safe storage.
28

3.3.3 Data Security

All user data is stored in MongoDB Atlas with secure connection strings hidden in environment variables. The frontend avoids storing sensitive passwords and keeps login tokens only while the session is active.

3.4 Maintainability

The project separates HTML, CSS, and JavaScript so it is easier to modify. Code is organized into logical functions and server routes.

3.5 Modifiability
